# Case 008 — 케이스 양식 (결정 케이스)

**TL;DR.** ChromaDB 자동 컨텍스트 주입이 미구현인 현재, 케이스 본문 8섹션 강제는 *미래 인프라를 가정한 글쓰기*다. 검색 표면은 사실상 인덱스 한 줄이며, 길어진 본문은 박제 비용만 늘리고 가치 회수는 인프라 도입까지 지연된다. 양식에 **TL;DR 필드를 강제**하고 **본문 섹션은 선택**으로 전환한다 — 본 케이스 자체가 첫 적용.

## 1. 문제 (사실)

- 현재 케이스 본문 평균 89줄 (000~007). 가장 긴 case #007은 128줄
- ChromaDB 미구현 상태에서 본문 8섹션을 *사람이 다시 읽는 경로*는 없음
- 박제의 두 가치 — (i) 결정 책임 단일 출처, (ii) 미래 자동 컨텍스트 — 중 (ii)는 약속 어음
- (i)는 frontmatter + TL;DR + 인덱스 한 줄로 99% 충족 가능
